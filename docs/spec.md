# 象与骑象人 — 互动叙事游戏 + 情绪消解智能体

## 项目功能规格说明书（spec.md）

> **项目定位**：基于积极心理学经典著作《象与骑象人》（Jonathan Haidt 著），开发两个互补产品——互动叙事游戏与情绪消解智能体，帮助用户在沉浸式体验中理解情绪机制，并在日常生活中实践情绪转化。

---

## 一、项目概述

本项目包含两个核心产品：

| # | 产品 | 定位 |
|---|------|------|
| 1 | **互动叙事游戏** | 玩家扮演"骑象人"，与代表情绪本能的"象"互动，通过选择推进故事，体验情绪与理性的博弈 |
| 2 | **情绪消解智能体** | 基于书中理论方法，帮助用户识别和转化消极情绪的 AI 对话助手（本地规则匹配版） |

---

## 二、技术栈

| 类别 | 选型 |
|------|------|
| 前端 | 纯 HTML5 + CSS3 + JavaScript（原生，不使用任何框架） |
| 运行环境 | 所有代码在浏览器中直接运行，无需构建工具 |
| 数据存储 | 浏览器 `localStorage` |
| 后端 | 不需要后端服务器（本地规则匹配版智能体） |

## 四、模块一：互动叙事游戏

### 4.1 故事引擎（`game/js/engine.js`）

**职责**：驱动故事节点的加载与跳转。

| 项目 | 说明 |
|------|------|
| 输入 | 当前故事节点 ID |
| 输出 | 节点文本 + 可选选项列表 |
| 核心逻辑 | 根据当前节点 ID 查找故事数据，返回对应内容 |
| 功能支持 | 节点跳转、条件分支 |

### 4.2 选择系统

**职责**：处理玩家选择，驱动故事推进与情绪变化。

每个选项关联以下属性：

| 属性 | 类型 | 说明 |
|------|------|------|
| `nextNodeId` | `string` | 下一故事节点 ID |
| `emotionImpact` | `number` | 情绪影响值 |
| `type` | `string` | 选择类型：`elephant`（象）/ `rider`（骑象人）/ `harmony`（和谐） |

**行为规则**：
- 选择后更新情绪状态，跳转到下一节点
- 记录所有选择历史（用于存档/复盘）

### 4.3 情绪追踪系统（`game/js/emotion.js`）

**职责**：实时追踪和展示玩家的情绪状态。

维护三个核心数值：

| 数值 | 范围 | 说明 |
|------|------|------|
| 象的力量值 | 0–100 | 代表情绪本能的强度 |
| 骑象人控制值 | 0–100 | 代表理性控制的能力 |
| 和谐度 | 0–100 | 自动计算：`100 - |象的力量值 - 骑象人控制值|` |

**更新规则**：
- 每次选择后根据选项类型更新数值
- 背景颜色随情绪状态动态变化：
  - 和谐度高 → 偏暖色（橙、黄）
  - 和谐度低 → 偏冷色（蓝、灰）

### 4.4 故事数据（`game/js/story.js`）

**职责**：以 JSON 格式存储全部故事节点。

**节点数据结构**：

```json
{
  "id": "chapter1_node1",
  "chapter": "第一章：遇见你的象",
  "text": "故事文本内容...",
  "options": [
    {
      "text": "选项文本",
      "type": "elephant | rider | harmony",
      "emotionImpact": 10,
      "nextNodeId": "chapter1_node2"
    }
  ]
}
```

**字段说明**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | `string` | 是 | 节点唯一标识 |
| `chapter` | `string` | 是 | 所属章节名 |
| `text` | `string` | 是 | 故事文本内容 |
| `options` | `array` | 是 | 选项数组（结尾节点可为空） |
| `options[].text` | `string` | 是 | 选项显示文本 |
| `options[].type` | `string` | 是 | 选择类型 |
| `options[].emotionImpact` | `number` | 是 | 情绪影响值 |
| `options[].nextNodeId` | `string` | 是 | 下一节点 ID |

### 4.5 故事大纲（6 章）

| 章节 | 章节名 | 核心主题 |
|------|--------|----------|
| 第一章 | 遇见你的象 | 认识情绪本能 |
| 第二章 | 象的暴走 | 体验情绪失控 |
| 第三章 | 骑象人的挣扎 | 理性控制的局限 |
| 第四章 | 对话与理解 | 开始倾听象的声音 |
| 第五章 | 合作之路 | 学会与象共处 |
| 终章 | 象骑合一 | 三种结局分支 |

### 4.6 结局系统

**结局判定逻辑**：

| 条件 | 结局名称 | 描述 |
|------|----------|------|
| 和谐度最高 | **象骑合一**（最佳结局） | 象与骑象人和谐合作，彼此支持 |
| 象的力量值 > 骑象人控制值 | **被象驱使** | 情绪失控，随波逐流 |
| 骑象人控制值 > 象的力量值 | **压抑崩溃** | 过度理性，最终崩溃 |

**结局展示**：
- 每个结局提供对应的心理学启示解读
- 引导玩家反思自身情绪管理模式

### 4.7 UI 界面（`game/index.html` + `game/css/style.css` + `game/js/ui.js`）

| 功能 | 说明 |
|------|------|
| 打字机效果 | 故事文本逐字显示，增强沉浸感 |
| 选项卡片 | 卡片式布局，鼠标悬停有视觉反馈效果 |
| 动态背景 | 背景颜色根据情绪状态实时变化 |
| 情绪数值条 | 顶部显示三个数值条：象的力量、骑象人控制、和谐度 |
| 存档/读档 | 支持游戏进度保存与恢复 |

## 五、模块二：情绪消解智能体

### 5.1 对话系统（`agent/js/chat.js`）

**职责**：管理多轮对话流程。

| 功能 | 说明 |
|------|------|
| 多轮对话 | 支持连续对话交互 |
| 持久化存储 | 对话历史通过 `localStorage` 持久保存 |
| 重新开始 | 支持清空当前对话，重新开始 |
| 界面样式 | 聊天窗口样式（左用户 / 右 AI） |

### 5.2 情绪识别（`agent/js/emotion.js`）

**职责**：基于关键词匹配识别用户当前情绪类型。

**情绪类型及关键词映射**：

| 情绪类型 | 关键词 |
|----------|--------|
| 焦虑 | 焦虑、紧张、担心、害怕、不安、压力 |
| 悲伤 | 悲伤、难过、伤心、失落、哭、痛苦 |
| 愤怒 | 愤怒、生气、烦、讨厌、不公平、凭什么 |
| 迷茫 | 迷茫、不知道、困惑、方向、意义、怎么办 |
| 孤独 | 孤独、寂寞、没人、一个人、不被理解 |
| 疲惫 | 疲惫、累、倦、无力、撑不住、没劲 |

**行为规则**：
- 每种情绪关联对应的理论方法（见 5.3）
- 支持多情绪同时识别（取匹配度最高的情绪类型）

### 5.3 理论方法库（`agent/js/theory.js`）

**职责**：基于《象与骑象人》的五大理论方法，提供结构化的引导方案。

#### 五大方法总览

| 方法名称 | 适用情绪 | 核心理念 |
|----------|----------|----------|
| 冥想训练 | 焦虑 | 引导观察念头而不评判 |
| 认知重构 | 悲伤、愤怒 | 改变对事件的解释方式 |
| 利他行动 | 孤独 | 通过帮助他人获得幸福感 |
| 进展感 | 迷茫、疲惫 | 创造小胜利积累动力 |
| 社会连接 | 孤独 | 建立深层人际关系 |

#### 每种方法包含的内容

| 内容项 | 说明 |
|--------|------|
| 方法名称 | 方法的中文名称 |
| 方法描述 | 方法的简要说明 |
| 适用情绪类型 | 该方法适用的情绪类型列表 |
| 引导话术模板 | 3–5 轮对话模板，用于引导用户进行练习 |
| 具体行动建议 | 分为三个层级（见 5.5） |

### 5.4 对话流程设计

智能体对话遵循以下六步流程：

```
┌─────────────────────────────────────────────┐
│  ① 倾听与共情                                │
│     承认"象"的感受是真实的                     │
│                    ↓                        │
│  ② 情绪识别                                  │
│     判断情绪类型和强度                         │
│                    ↓                        │
│  ③ 方法匹配                                  │
│     选择最合适的理论方法                       │
│                    ↓                        │
│  ④ 引导练习                                  │
│     带领用户进行具体练习                       │
│                    ↓                        │
│  ⑤ 行动建议                                  │
│     给出 1 个可执行的行动                      │
│                    ↓                        │
│  ⑥ 复查确认                                  │
│     确认情绪是否有变化                         │
└─────────────────────────────────────────────┘
```

| 步骤 | 名称 | 说明 |
|------|------|------|
| ① | 倾听与共情 | 承认"象"的感受是真实的，表达理解 |
| ② | 情绪识别 | 判断情绪类型和强度 |
| ③ | 方法匹配 | 根据情绪类型选择最合适的理论方法 |
| ④ | 引导练习 | 按照方法模板带领用户进行具体练习 |
| ⑤ | 行动建议 | 给出 1 个具体可执行的行动 |
| ⑥ | 复查确认 | 确认情绪是否有变化，决定是否需要进一步引导 |

### 5.5 行动建议系统

**职责**：每次对话生成 1 个具体可执行的行动。

行动分为三个层级：

| 层级 | 时间范围 | 说明 |
|------|----------|------|
| 即时行动 | 1 分钟内 | 可立即执行的小行动（如深呼吸 3 次） |
| 短期行动 | 1 小时内 | 需要少量时间投入的行动（如散步 10 分钟） |
| 日常练习 | 持续进行 | 可纳入日常习惯的练习（如每日感恩日记） |

**交互规则**：
- 行动完成后可标记为"已完成"
- 完成的行动积累"进展感"，激励用户持续练习

### 5.6 情绪日志

**职责**：记录和可视化用户的情绪变化。

| 功能 | 说明 |
|------|------|
| 情绪记录 | 记录每次对话的情绪状态（类型、时间、强度） |
| 趋势查看 | 可查看情绪变化趋势图 |
| 情绪报告 | 提供阶段性情绪报告总结 |

### 5.7 安全机制

**职责**：保障用户安全，明确工具边界。

| 规则 | 说明 |
|------|------|
| 危机检测 | 检测到危机关键词（自杀、不想活、结束生命等）时，立即显示专业求助热线 |
| 不做诊断 | 不做医学诊断或药物建议 |
| 免责提示 | 明确提示本工具不替代专业心理咨询 |

**危机关键词示例**：自杀、不想活、结束生命、活着没意思、想死、自我伤害等。

### 5.8 UI 界面（`agent/index.html` + `agent/css/style.css` + `agent/js/ui.js`）

| 功能 | 说明 |
|------|------|
| 聊天窗口 | 左侧显示用户消息，右侧显示 AI 回复 |
| 情绪标识 | 顶部显示当前识别的情绪类型 |
| 侧边栏 | 显示情绪日志和情绪变化趋势图 |
| 输入区域 | 底部输入框，支持发送消息 |

## 八、设计原则

1. **心理学准确性**：所有理论方法严格基于《象与骑象人》原著
2. **无障碍使用**：纯前端实现，无需注册、无需网络（除首次加载）
3. **数据隐私**：所有数据存储在用户本地浏览器，不上传任何服务器
4. **渐进式引导**：游戏通过叙事自然引入理论，智能体提供日常实践支持
5. **安全第一**：内置危机检测机制，必要时引导用户寻求专业帮助
